# Chinese-Lexicon-Hub / Sinolingua Guide

Коротко: монорепозиторий (pnpm workspace) с артефактами мобильного приложения для изучения китайского языка (Expo / React Native), API-сервера (Express) и вспомогательных пакетов. Проект служит учебным/демонстрационным приложением для студентов китайского: словари, идиомы, сленг, произношение, диалоговые практики и культурные заметки.

О проекте
=========
Sinolingua Guide — мобильное приложение-справочник для изучающих китайский язык. Данные (наборы слов, идиом, сленга, диалогов, культурных заметок и пр.) встроены как TypeScript-датасеты в пакеты приложения. Репозиторий оформлен как pnpm-monorepo с несколькими пакетами/артефактами:
- artifacts/sinolingua-guide — Expo/React Native мобильное приложение (UI, данные, экранные компоненты);
- artifacts/api-server — Express 5 API-сервер для обслуживания данных (локально/демо);
- artifacts/mockup-sandbox — окружение для быстрого прототипирования UI/макетов;
- другие пакеты (api-схемы/зогенераторы, db и т.д.) — вспомогательные части монорепозитория.

Ключевые возможности
===================
- 成语 (Chengyu): набор классических четырехзнаковых идиом с pinyin, переводами, значениями и примерами.
- 网络用语 (Slang & Abbreviations): современные интернет-выражения с происхождением и примерами.
- 流行文化 (Pop Culture): фразы из медиа и мемов с культурным контекстом.
- 发音 (Pronunciation): руководство по тонам, минимальные пары, проблемные звуки и тон-сандхи.
- 对话练习 (Dialogues): интерактивные сценарии с multiple-choice и ветвлением.
- Культурный гид: записи об этикете, праздниках, обычаях, мы́слях о "лице" (面子), и пр.
- Полнотекстовый поиск и фильтры (по pinyin, словам, тегам, сложности и т.д.)

Структура репозитория (основные элементы)
=========================================
(корневой уровень)
- package.json — главный workspace package (scripts, devDeps).
- pnpm-workspace.yaml — конфигурация workspace.
- tsconfig.base.json, tsconfig.json — TypeScript конфигурации.
- replit.md — заметки и инструкция для запуска в Replit (обзор проекта и ключевых команд).
- artifacts/ — содержимое артефактов:
  - artifacts/sinolingua-guide/ — мобильное приложение (Expo)
    - package.json — зависимости (expo, react-native, библиотеки UI и т.д.)
    - app/, components/, data/, assets/, server/, scripts/ и пр.
  - artifacts/api-server/ — сервер
    - build.mjs — сборка esbuild/preset (бандлинг server -> dist)
    - package.json — зависимости сервера (express, pino, drizzle-orm и пр.)
    - src/ — исходники сервера (точка входа src/index.ts)
  - artifacts/mockup-sandbox/ — санбокс макетов UI
- scripts/ — утилиты рабочего пространства (typecheck и пр.)
- attached_assets/, screenshots/ — вспомогательные каталоги

Технологический стек
====================
- Менеджер пакетов и монорепозиторий: pnpm workspaces
- Node.js: ориентировочно версия 24 (рекомендуется использовать Node 24+)
- TypeScript: ~5.9.x
- Мобильное приложение: Expo / React Native (expo-router)
- Сервер: Express 5
- БД и ORM: PostgreSQL + Drizzle ORM (в проекте упоминается Drizzle)
- Валидация: Zod (zod v4)
- Генерация API: Orval (OpenAPI -> hooks + Zod)
- Сборка серверных артефактов: esbuild (в т.ч. кастомный скрипт build.mjs)
- Логирование: pino / pino-http
- Форматирование: Prettier

Установка (локально)
====================
1. Установите Node.js (рекомендуется v24) и pnpm (последняя версия).
2. Клонируйте репозиторий:
   git clone https://github.com/Astagfiru/Chinese-Lexicon-Hub.git
3. Перейдите в корень репозитория и установите зависимости:
   pnpm install
   (В репозитории настроен preinstall, который ожидает использование pnpm.)

Основные команды
================
- pnpm run typecheck — полная проверка типов по workspace.
- pnpm run build — выполнить typecheck и собрать все пакеты (через pnpm -r).
- pnpm --filter @workspace/api-server run dev — запустить API сервер в режиме разработки.
- pnpm --filter @workspace/sinolingua-guide run dev — запустить Expo для мобильного приложения.
- pnpm --filter @workspace/api-spec run codegen — регенерация API hooks и Zod-схем из OpenAPI (если пакет @workspace/api-spec присутствует/настроен).
- pnpm --filter @workspace/db run push — применить изменения схемы БД (dev only).

Запуск API-сервера (dev и prod)
===============================
- Dev:
  - В пакете artifacts/api-server есть script `dev`, который делает:
    export NODE_ENV=development && pnpm run build && pnpm run start
  - Локально: pnpm --filter @workspace/api-server run dev
- Сборка:
  - Скрипт сборки `build` в artifacts/api-server запускает `node ./build.mjs` — esbuild собирает `src/index.ts` в `dist/` (формат ESM, расширение .mjs). В сборке внешние (native) модули игнорируются (перечень externals в build.mjs).
- Запуск скомпилированного:
  - node --enable-source-maps ./dist/index.mjs

Мобильное приложение (Expo)
===========================
- Основная директория: artifacts/sinolingua-guide
- Скрипты:
  - dev — запускает Expo в режиме разработки. (В package.json dev-скрипт использует переменные окружения, полезные при запуске в Replit: EXPO_PACKAGER_PROXY_URL, EXPO_PUBLIC_DOMAIN, REPL_ID и т.п.)
  - build — локальная сборка front-end-ресурсов (скрипт `node scripts/build.js`).
  - serve — сервер для статических/превью артефакта.
- Запуск локально:
  pnpm --filter @workspace/sinolingua-guide run dev
- Примечание: при запуске Expo убедитесь, что у вас установлены Expo CLI и корректные версии React / React Native, описанные в package.json артефакта.

Структура данных и контента
==========================
- Основные данные (слова, фразы, примеры, диалоги) встроены в приложение в виде TypeScript-датасетов (папки `data/` в sinolingua-guide).
- Данные объединены и отформатированы с указанием источников (в т.ч. открытые лингвистические ресурсы: ChID, PETCI, Tencent NLP, LIVAC — см. примечания в replit.md).
- Поиск и фильтрация реализованы на стороне клиента (full-text search и фильтры по тегам/категориям).

Разработка базы данных и API-генерация
====================================
- В проекте упоминается Drizzle ORM и PostgreSQL для хранения данных на сервере.
- Для генерации API-клиентов и Zod-схем используется Orval (OpenAPI -> код).
- Команды:
  - pnpm --filter @workspace/api-spec run codegen — пересоздать клиентские хуки и Zod-схемы из OpenAPI.
  - pnpm --filter @workspace/db run push — применить миграции/обновления схемы (если настроено).

Сборка сервера (технические детали)
=================================
- Скрипт сборки `artifacts/api-server/build.mjs` использует esbuild для бандлинга серверного кода в ESM (.mjs) с source maps.
- Для обеспечения работы некоторых CJS-зависимостей и логгера pino используется плагин esbuild-plugin-pino и баннер, задающий globalThis.require и __dirname/__filename.
- В build.mjs есть большой список external-пакетов (native, нативные библ., базы данных, heavy libs) — они не бандлятся.

Replit (заметки и запуск)

Полезные примечания и ограничители
=================================
- Это монорепо: некоторые пакеты являются workspace:* зависимостями (локальные пакеты).
- В сборке сервера многие native/тяжелые пакеты исключены (external) — при необходимости их нужно устанавливать и конфигурировать отдельно.
- Некоторые package.json содержат placeholder/`catalog:` версии — убедитесь, что у вас правильная конфигурация registry или локальных ссылок при установке.
- Данные в приложении позиционируются как открытые/образовательные и основаны на общедоступных источниках.
